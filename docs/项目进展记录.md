## 一、项目概况

本项目基于 Spring Boot + Vue 3 + MySQL 单体架构，围绕用户注册/登录、购物车、订单流转、管理员后台等电商核心流程展开开发与测试。Excel 任务表中共规划 12 项任务（T1-T12），总工作量 **70 人时**，覆盖开发、单元测试、用例设计、手工测试、自动化测试、性能测试及文档统筹。

> **重要说明**：原始 Excel 文件仅记录计划与分工，未记录实际完成状态。本进展记录通过核对代码库、测试报告及实际运行结果得出。

## 四、偏差分析

### 4.1 整体偏差

| 维度      | 计划                | 实际                                 | 偏差         |
| :------ | :---------------- | :--------------------------------- | :--------- |
| 核心功能代码  | 完成 T1-T3          | 已完成                                | 无偏差        |
| 单元测试    | T1-T3 达到覆盖率 > 80% | 后端 117 用例 100% 通过；前端 62 用例 100% 通过 | 优于计划       |
| 测试用例设计  | T4、T5 完成          | 均未完成                               | **严重滞后**   |
| 手工测试执行  | T6 完成             | 未执行                                | **严重滞后**   |
| 接口自动化   | T7 完成             | 脚本为空                               | **严重滞后**   |
| E2E 自动化 | T8 脚本通过           | 脚本存在但运行失败                          | **50% 滞后** |
| 性能测试    | T9-T11 完成         | 均未完成                               | **严重滞后**   |
| 文档报告    | T12 完成            | 仅单元测试报告完整                          | **60% 滞后** |

### 4.2 偏差根因

1. **重开发、轻测试**：T1-T3 开发及单元测试已高质量完成，但测试设计、手工执行、自动化脚本、性能测试等任务尚未落地。
2. **文档与交付物不同步**：测试用例、自动化脚本、性能报告等文件已创建但内容为空，说明占位意识存在但执行未跟上。
3. **E2E 脚本运行阻塞**：登录接口存在 JSON 解析异常（后端日志显示 `Unexpected character ('u')`），导致 Playwright 购物流程用例无法进入首页，需排查前后端请求格式与测试数据准备。
4. **性能测试未启动**：T9-T11 无脚本、无报告、无执行痕迹，是当前最大风险点。

### 6.2 后续行动建议

1. **立即补齐测试用例**（T4、T5）：按黑盒方法编写买家/管理员业务用例，重点覆盖购物车 1-99 边界、收货信息域、密码格式、管理员权限、课程标识检查点。
2. **修复 E2E 登录失败**（T8）：
   - 确认后端登录接口接收的 Content-Type 与请求体格式一致；
   - 在测试数据库中预置 `testuser / Test1234` 账号；
   - 确认前端 Vite 代理将 `/api` 转发至后端 8080 端口。
3. **启动接口自动化**（T7）：基于 Python + requests 覆盖注册/登录/购物车/订单等核心接口的边界与异常场景。
4. **启动性能测试**（T9-T11）：使用 JMeter 或 Locust 设计 50/200 并发场景，验证防超卖、90% 响应时间、30 分钟稳定性及资源监控指标。
5. **完成最终报告**（T12）：汇总手工、自动化、性能数据，按课程模板输出完整实践报告。
